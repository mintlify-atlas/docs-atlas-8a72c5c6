`@phake/mcp` is a TypeScript library for building [MCP (Model Context Protocol)](https://modelcontextprotocol.io) servers that run on Cloudflare Workers and Node.js. It handles routing, authentication, token storage, and the MCP protocol so you can focus on writing tools.

## Key features

- **Type-safe tools with Zod** — define input and output schemas using Zod; TypeScript types are inferred automatically with no manual casting
- **Multiple auth strategies** — OAuth 2.1 PKCE, Bearer tokens, API keys, custom headers, or no auth — all configured through environment variables
- **KV-backed token storage** — Cloudflare KV with AES-256-GCM encryption at rest, plus in-memory fallback for local development
- **SSE + JSON-RPC transport** — full MCP protocol support over Server-Sent Events and JSON-RPC 2.0
- **Built-in tools** — `health` and `echo` tools included by default, ready to test your server without writing any code

## When to use `@phake/mcp`

Use `@phake/mcp` when you want to:

- Deploy an MCP server to Cloudflare Workers with minimal boilerplate
- Add OAuth or API key authentication to your MCP tools without implementing the protocol yourself
- Define tools with validated inputs and structured outputs using Zod schemas
- Run the same tool definitions on both Cloudflare Workers and Node.js

If you are building a simple local stdio server for a single client, the raw [`@modelcontextprotocol/sdk`](https://github.com/modelcontextprotocol/typescript-sdk) may be sufficient. `@phake/mcp` is best suited for deployable, multi-client servers where auth and storage matter.

## Requirements

- **Bun 1.x** or **Node.js 20+**
- **Wrangler CLI** — required for Cloudflare Workers deployments
- **Cloudflare account** with Workers and KV access — required for the `worker` adapter

## Package exports

| Export | Import path | Description |
|--------|-------------|-------------|
| Core | `@phake/mcp` | `defineTool`, `createMCPServer`, and all helpers |
| Cloudflare Workers adapter | `@phake/mcp/runtime/worker` | Cloudflare Workers adapter internals |
| Node.js adapter | `@phake/mcp/runtime/node` | Node.js adapter internals (experimental) |

Most projects only need to import from `@phake/mcp` directly.
