---
title: "Quickstart"
description: "Deploy your first MCP server to Cloudflare Workers in minutes."
---

<Steps>
  <Step title="Install the package">
    <CodeGroup>

    ```bash npm
    npm install @phake/mcp
    ```

    ```bash bun
    bun add @phake/mcp
    ```

    ```bash pnpm
    pnpm add @phake/mcp
    ```

    </CodeGroup>
  </Step>

  <Step title="Create the KV namespace">
    Phake MCP stores tokens and sessions in Cloudflare KV. Create the namespace with Wrangler:

    ```bash
    wrangler kv namespace create TOKENS
    ```

    Copy the namespace ID printed in the output — you'll need it in the next step.
  </Step>

  <Step title="Bind the KV namespace in your Wrangler config">
    Add the namespace binding to your Wrangler configuration file. Replace `<your-kv-namespace-id>` with the ID from the previous step.

    <CodeGroup>

    ```toml wrangler.toml
    [[kv_namespaces]]
    binding = "TOKENS"
    id = "<your-kv-namespace-id>"
    ```

    ```jsonc wrangler.jsonc
    {
      "kv_namespaces": [
        {
          "binding": "TOKENS",
          "id": "<your-kv-namespace-id>"
        }
      ]
    }
    ```

    </CodeGroup>
  </Step>

  <Step title="Generate an encryption key">
    Tokens are encrypted at rest with AES-256-GCM. Generate a 32-byte key:

    <CodeGroup>

    ```bash openssl
    openssl rand -base64 32 | tr '+/' '-_' | tr -d '='
    ```

    ```bash node
    node -e "const {randomBytes}=require('crypto'); console.log(randomBytes(32).toString('base64url'))"
    ```

    </CodeGroup>

    Save the output — you'll use it in the next step.
  </Step>

  <Step title="Set the encryption key">
    **Production** — add the key as a Wrangler secret:

    ```bash
    wrangler secret put RS_TOKENS_ENC_KEY
    # paste the generated key when prompted
    ```

    **Local development** — add the key to `.dev.vars` in your project root:

    ```ini .dev.vars
    RS_TOKENS_ENC_KEY=<your-generated-key>
    ```
  </Step>

  <Step title="Create your Worker">
    Create an entry point file (e.g., `src/index.ts`) with your tools and server:

    ```typescript src/index.ts
    import { z } from "zod";
    import { defineTool, createMCPServer } from "@phake/mcp";

    const greetTool = defineTool({
      name: "greet",
      title: "Greet User",
      description: "Returns a greeting for the given name",
      inputSchema: z.object({
        name: z.string().describe("Name to greet"),
      }),
      handler: async (args) => {
        return { message: `Hello, ${args.name}!` };
      },
    });

    const server = createMCPServer({
      adapter: "worker",
      tools: [greetTool],
    });

    export default server;
    ```
  </Step>

  <Step title="Deploy to Cloudflare Workers">
    ```bash
    wrangler deploy
    ```

    Wrangler prints your Worker URL after a successful deploy (e.g., `https://my-mcp-server.your-account.workers.dev`).
  </Step>

  <Step title="Connect a client">
    Point your MCP client at the deployed URL. Use the `/mcp` path for the SSE endpoint.

    **Claude Desktop** — add to `~/Library/Application Support/Claude/claude_desktop_config.json` (macOS) or `%APPDATA%\Claude\` (Windows):

    ```json claude_desktop_config.json
    {
      "mcpServers": {
        "my-server": {
          "url": "https://my-mcp-server.your-account.workers.dev/mcp"
        }
      }
    }
    ```

    **Cursor / VSCode** — add to `mcp.json` in your project root or `.vscode/mcp.json`:

    ```json mcp.json
    {
      "mcpServers": {
        "my-server": {
          "url": "https://my-mcp-server.your-account.workers.dev/mcp"
        }
      }
    }
    ```
  </Step>
</Steps>

<Note>
  Every Phake MCP server includes two built-in tools by default:

  - **`echo`** — accepts `{ message, uppercase? }` and returns `{ echoed, length }`. Useful for verifying your server is reachable.
  - **`health`** — accepts `{ verbose? }` and returns `{ status, timestamp, runtime, uptime? }`. Available at `GET /health` as well as through the MCP protocol.

  You do not need to register these tools yourself.
</Note>
